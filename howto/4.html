Final Considerations

Using the concepts demonstrated in these demos and a knowledge of the Riot Games API, we can now put together a functional, if basic, match history website. In this last section, I would like to go a small bit further into the details of either the non-technical aspects of design or some of the things that I glossed over in the demos without mentioning.

Design Decisions

As always, the first step to making a website is considering how it should be used. In the descriptions for the demos, I dictated vague ideas of how the website would be used, but here I would like to specifically describe the user's flow through the website. There are two pages on the site:

  • Summoner Match History

    The top of the page is a search box for summoner (player) names. When you put in a name and execute the search, the rest of the page asynchronously fills with results that compose the player's match history. Each result is a match which includes basic, high level stats for the player's performance in that match. Each result also has a link to the Detailed Match Information page that describes it.

  • Detailed Match Iformation

    This page should only ever be accessed from the Match History page or a direct link that was ultimately accquired through the Match History page. This is because this page will take a request parameter, the match ID, and populate the information using that. THe match information will include details of who was in the match, how they did, and which team won. Each participant's information section will include a link to their match history page.

This is a pretty basic but standard design for a match history site. It is easy to find a specific player, provided you have their name. It is possible to navigate through their matches, find other people in their matches, and meander around that way.

API Region

The first detail that I glossed over was that all of the API calls include a {region} parameter. For the most part, this selects which set of Riot's game servers the information is pulled from. Additionally, there is another hostname for each region as well as some references to platform IDs, each of which define the specific set of servers that are queried against. In all of the demos, I used the region na, North America. There is a whole list of regions available here if you are interested in another one.

Demo Functionality

Like the official site mentions many, many times, you want to hide your API key. I'll leave the explaining why to Riot, but I would like to describe how the demos work. Each demo with a button invokes a JavaScript call to do its work. The calls use JQuery to easily pull information from and write information to the page. Perhaps the most important part, however, is how we asynchronously access the Riot Games API.

It would be pretty simple to just make an asynchronous request to the API itself, but that would mean my API key is directly visible in the JavaScript, which is copied to and runs on the client's browser. Instead, I made a simple PHP wrapper which takes an arbitrary API call string, tacks it onto the hostname, and then appends my API key to the end. This hides the API key itself, but a random internet-goer could access the URL directly and execute arbitrary API calls. To preven this, the wrapper also makes sure that the request to the PHP page is coming from the same server as the PHP page itself is running on, preventing arbitrary access.

Closing Thoughts

Thanks for checking out the how-to guide. There's still tons of other things that would be needed for a truly good match history site, but they don't directly involve the Riot Games API, so I did not include them here. The first things that I think of are data aggregation for tracking trends, dealing with media (ie. images as pulled from Riot's Data Dragon), and all sorts of other stuff. Even using the API, there are many features still being added or available only to established applications. For my Final, I intend to use a fair amount of this work as well as pull in the media and maybe some data to make a (hopefully not crap) match history site. See you there!
