First Step

Player History

The first thing we need to do is provide users with some way to find the game they're interested in. The best way to find a match is to search on a participating player, then select the specific match from a list of their played matches.

To do this, we use the matchhistory-v2.2 request which returns a list of matches in which a player participated, however it takes a summonerId as input. This id is effectively a unique number associated with a player and can be obtained from the summoner-v1.4 method with a by-name request. Both of these requests return JSON containing the data we want. We get the summoner Id from the first request and then plug it into the second.

Check the demos below to see how this all looks. Try Zirene for a summoner name.
